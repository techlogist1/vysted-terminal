# OpenBB subprocess — its own venv with OpenBB's strict pins.
#
# OpenBB-core 1.6.9 requires fastapi <0.129 and uvicorn <0.41 strictly, which
# are incompatible with the main Vysted sidecar's pins (fastapi 0.136, uvicorn
# 0.46). Isolating OpenBB into its own process+venv keeps the main sidecar's
# dependency tree clean — the Tier 2 path per plan §A2 + BLOCKERS-C.md.
#
# This subprocess exposes a minimal FastAPI service the main sidecar's
# `services.openbb_provider` proxies through; the main sidecar starts the
# subprocess lazily on first OpenBB request and kills it via the standard
# stdin-EOF watchdog when the Tauri core drops the parent.
fastapi==0.128.8
uvicorn[standard]==0.40.0
pydantic==2.13.4
openbb-core==1.6.9
openbb-equity==1.6.1
openbb-economy==1.6.1
